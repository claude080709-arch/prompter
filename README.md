# Prompter

목표를 입력하면 AI가 최적의 전문가 페르소나를 추천하고, 그 전문가 관점에서 즉시 사용 가능한 프롬프트를 생성해주는 도구.

🔗 **라이브 링크:** [Prompter](https://prompter-lovat.vercel.app/)

---

## 1. 왜 만들었나

좋은 AI 결과물을 얻으려면 프롬프트를 잘 써야 하는데, 어떤 전문가 관점으로 접근해야 할지 막막한 경우가 많다. "마케터처럼 생각해줘"보다 구체적인 페르소나와 맥락이 담긴 프롬프트가 필요했다. 목표만 입력하면 전문가 선택부터 프롬프트 생성까지 자동화하고 싶었다.

---

## 2. 주요 기능

| 기능 | 설명 |
|------|------|
| 전문가 추천 | 목표 입력 → AI가 최적 전문가 3인 자동 추천 |
| 전문가 선택 | 추천된 전문가 카드 중 원하는 것 선택 |
| 리롤(재추천) | 피드백 입력 → 이전 전문가 + 원래 목표 + 피드백 반영해 새 전문가 재추천 |
| 프롬프트 생성 | 선택한 전문가 관점의 한국어/영어 프롬프트 즉시 생성 |
| 복사 | 생성된 프롬프트 원클릭 복사 |

---

## 3. 기술 스택 및 선택 이유

- **HTML + CSS + JS (단일 파일)** — 프레임워크 없이 빠른 로딩, 복잡한 빌드 불필요
- **Groq API (llama-3.3-70b-versatile)** — 무료 tier에서 고성능 LLM 사용 가능, 응답 속도 빠름
- **Vercel Serverless Functions** — API 키를 서버에서 관리, 클라이언트에 노출 없음
- **2페이지 구조 (index.html + app.html)** — 랜딩과 앱 분리로 첫인상과 기능 집중

---

## 4. 아키텍처

```
index.html (랜딩)
    └── app.html (앱)
            └── /api/groq  ── Groq API (llama-3.3-70b-versatile)
                               ├── 전문가 추천 (JSON 구조화 출력)
                               ├── 리롤 (목표 + 이전 전문가 + 피드백 컨텍스트)
                               └── 프롬프트 생성 (한국어/영어)
```

---

## 5. 어려웠던 점

**① 한자 노출 버그**
LLM이 한국어 응답 중간에 한자(漢字)를 섞어 출력하는 현상 발생. 시스템 프롬프트에 `NO Chinese characters (한자)` 규칙을 명시적으로 추가하고, 클라이언트 측에서도 CJK 유니코드 범위 정규식으로 한자를 제거하는 이중 방어 처리로 해결.

**② 리롤 컨텍스트 설계**
단순 재요청 시 이전과 비슷한 전문가가 반복 추천되는 문제. 원래 목표 + 이미 추천된 전문가 목록 + 사용자 피드백을 하나의 컨텍스트로 묶어 전달하는 방식으로 해결. 이전 결과를 명시적으로 제외 조건으로 넣는 것이 핵심이었다.

**③ JSON 파싱 안정성**
LLM이 JSON 외에 설명 텍스트를 함께 출력하는 경우가 있음. JSON 블록만 정확히 추출하는 파서를 별도 구현해 처리.

## 6. 로컬 실행 가이드

`.env` 파일을 프로젝트 루트에 생성:

```env
GROQ_API_KEY=gsk_...
```

API 키 발급: console.groq.com → API Keys

```bash
npm i -g vercel
vercel dev
```
→ `localhost:3000` 접속
